# Quick Research (L0)

Fast lightweight research for simple questions. No planner, no reflection, no multi-agent — just a quick multi-source search with a cited answer.

**Position in ladder:** L0 — the lightest tier. When in doubt between this and `/research`, prefer `/research`.

## When to Use

- Fact checks: "какая последняя версия Next.js", "когда вышел Bun 2.0"
- Simple lookups: "что такое edge runtime", "кто автор библиотеки X"
- Yes/no questions with supporting evidence
- Any question the user frames as "быстро", "коротко", "кратко"

**Do NOT use for:**
- Comparisons (use `/deep-research`)
- Strategic decisions (use `/expert-research`)
- Topics requiring depth (use `/research` or higher)

## Budget

- Time: ~10–15 sec (Fireplexity-pattern: search+scrape+summary in one Firecrawl call)
- Firecrawl credits: ~5–10 (1 search + scrape with summary on 5 pages)
- Tavily credits: ~2 (1 search × 10 results = 2 credits)
- Sources read: 3–5
- Output: 2–4 paragraphs with inline citations

## Pipeline

```
v0.9.0+ Fireplexity-pattern (Perplexity replacement):
  1. SEARCH+SCRAPE — one firecrawl search with --scrape --scrape-formats markdown,summary
                     in parallel with mcp__tavily__tavily_search (diversity provider)
  2. SYNTH         — Claude reads pre-scraped markdown+AI-summary, writes 2-4 para
                     answer with inline [N] citations from the merged URL set

Total wall time: ~10-15 sec (one search round + Claude synthesis).
No separate scrape step needed — Firecrawl returns content in the search response.
```

## Step 1: SEARCH+SCRAPE (Fireplexity-pattern, parallel)

Run these TWO calls in PARALLEL in a single message — two independent providers give a
broader top-result set than either alone:

```bash
# Firecrawl: search + scrape + AI summary in ONE call
firecrawl search "<query>" \
  --limit 5 \
  --scrape \
  --scrape-formats markdown,summary \
  --only-main-content \
  --json
```

```
mcp__tavily__tavily_search with:
  query: <query>
  max_results: 5
  search_depth: "basic"
  topic: "general"     # use "news" if query is freshness-sensitive
```

For Russian-language topics: add one extra search in Russian (run in the same parallel
batch — three calls instead of two).

For freshness-sensitive queries ("latest", "2026", "what's new"), add `--tbs qdr:m`
to Firecrawl and `time_range: "month"` to Tavily.

> **Persistence note.** Tavily MCP responses live in conversation context only. If the
> answer needs to be reproducible / auditable, write the Tavily response JSON to disk
> via the Write tool. For L0 quick mode this is **optional** — L0 doesn't write artifacts.

## Step 2: SYNTH (one-pass synthesis)

Firecrawl already returned **markdown + AI-summary per URL** in the search response —
no separate scrape step is needed (this is the time-saver vs the old 3-step flow).

Steps:

1. Merge URL sets from Firecrawl + Tavily, deduplicate by URL
2. Pick the 3–5 most authoritative results (official docs > recent tech > community)
3. Read the `summary` field for each picked URL (already AI-generated by Firecrawl)
4. For URLs that came from Tavily only (no Firecrawl summary), either rely on Tavily's
   `content` snippet or — if the URL looks load-bearing — do one extra
   `firecrawl scrape "<url>" --only-main-content` (rare for L0)
5. Synthesize: 2–4 paragraphs with inline `[N]` citations

**Source priority:**
1. Official docs / primary sources
2. Recent articles (2025–2026)
3. Authoritative tech sources
4. Skip: SEO spam, affiliate pages, paywalled content

## Step 3: ANSWER

Format the response as:

```
**[Direct answer in 1-2 sentences]**

[2-3 supporting paragraphs with inline citations [1], [2], [3]]

---
**Sources:**
1. [Title](URL)
2. [Title](URL)
3. [Title](URL)
```

## Rules

- **Always cite** — every factual claim needs [N] reference
- **Direct answer first** — no preamble, no "let me research this"
- **Flag uncertainty** — if sources disagree, say so briefly
- **Freshness** — prefer 2025–2026 sources
- **Language** — match user's language (Russian / English)
- **No artifacts on disk** — this is a quick mode, don't save to `.firecrawl/research/`
- **No clarifying questions** — if query is ambiguous, pick most likely interpretation and answer

## Escalation

If during research you realize the question is more complex than expected (needs comparison, deep analysis, multiple perspectives), STOP and tell the user:

> "Этот вопрос серьёзнее чем quick-research. Рекомендую `/research` (L1) или `/deep-research` (L2). Продолжить quick или переключиться?"
